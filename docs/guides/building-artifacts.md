# Building Release Artifacts

Release artifacts are generated by `tools/build-artifacts.py`. The script runs
validation, packages source and Markdown documentation, optionally builds the
PDF manual, and writes SHA-256 checksums.

## Complete release build

```bash
python tools/build-artifacts.py
```

Required tools:

- Python 3.10 or newer
- GCC or Clang
- Doxygen
- Graphviz
- `make`
- TeX Live or MiKTeX with PDFLaTeX

The command performs:

1. native tests
2. public documentation audit
3. Markdown documentation packaging
4. Doxygen LaTeX generation
5. PDFLaTeX compilation
6. source packaging
7. SHA-256 checksum generation

## Markdown-only build

```bash
python tools/build-artifacts.py --markdown-only
```

This still produces the source archive and Markdown documentation archive but
does not require LaTeX.

## Documentation artifacts

```text
dist/
  eurorack-framework-<version>-markdown-docs.zip
  eurorack-framework-<version>-manual.pdf
```

The Markdown archive contains the maintained documentation sources from
`docs/manual`, `docs/guides`, `docs/architecture`, and `docs/release`, plus the
README.

The PDF combines the newcomer-oriented manual and the Doxygen API reference.

## Source archive

The source archive includes source, headers, tests, examples, configuration,
documentation sources, tools, licensing files, and CI configuration. Generated
documentation and local build caches are excluded.

## Reproducibility

ZIP entries are sorted and use normalized timestamps. Checksums verify transfer
integrity but are not a cryptographic publisher signature.
