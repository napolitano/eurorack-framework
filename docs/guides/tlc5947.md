# TLC5947 integration and startup behaviour

## Purpose

`eurorack-driver-tlc5947` controls one or more daisy-chained TLC5947 devices without dynamic memory.
The application owns both required buffers:

```cpp
std::array<std::uint16_t, 24> brightness{};
std::array<std::uint8_t, 36> frame{};
```

For `N` devices, allocate `24 * N` brightness entries and `36 * N` frame bytes.

## Safe electrical startup

The constructor immediately drives XLAT low and, when a BLANK/OE pin is supplied, keeps the outputs
blanked. `initialize()` always performs this sequence:

1. Assert BLANK.
2. Clear the caller-owned brightness buffer.
3. Transfer and latch a complete zero frame.
4. Execute the selected optional startup pattern.
5. Clear and latch a final zero frame.
6. Enable or retain blanking according to `enableAfterInitialize`.

A failed SPI transfer is never latched. Visible startup modes return with the outputs blanked when an
error occurs.

## Startup modes

### Silent

```cpp
eurorack::drivers::led::Tlc5947Config config{};
config.startupMode = eurorack::drivers::led::Tlc5947StartupMode::Silent;
```

This is the default and recommended production setting. No LED is intentionally illuminated.
No `DelayProvider` is required.

### Sequential

```cpp
eurorack::drivers::led::Tlc5947Config config{};
config.startupMode = Tlc5947StartupMode::Sequential;
config.startupBrightness = 0x3000;
config.startupStepMilliseconds = 30;
```

Each logical channel is illuminated individually in ascending order. This is useful for assembly
checks, channel mapping verification, and service diagnostics. A `DelayProvider` is mandatory.

### AllFlash

```cpp
eurorack::drivers::led::Tlc5947Config config{};
config.startupMode = Tlc5947StartupMode::AllFlash;
config.startupBrightness = 0x1800;
config.startupFlashMilliseconds = 100;
```

All channels illuminate together once and are then cleared. A `DelayProvider` is mandatory.

## Arduino Nano R3 example

```cpp
#include <array>
#include <eurorack/drivers/led/tlc5947.hpp>
#include <eurorack/platform/arduino/arduino_gpio.hpp>
#include <eurorack/platform/arduino/arduino_spi.hpp>
#include <eurorack/platform/arduino/arduino_time.hpp>

std::array<std::uint16_t, 24> brightness{};
std::array<std::uint8_t, 36> frame{};

eurorack::platform::arduino::ArduinoSpiBus spi;
eurorack::platform::arduino::ArduinoDigitalOutput latch(9);
eurorack::platform::arduino::ArduinoDigitalOutput blank(A0);
eurorack::platform::arduino::ArduinoTime time;

eurorack::drivers::led::Tlc5947Config config{
    8'000'000U,
    true,
    eurorack::drivers::led::Tlc5947StartupMode::Silent,
    0xFFFFU,
    35U,
    120U};

eurorack::drivers::led::Tlc5947 leds(
    spi, latch,
    brightness.data(), brightness.size(),
    frame.data(), frame.size(),
    1U, &blank, &time, config);
```

The example focuses on lifetime, storage, and initialization order.

## Runtime rules

- Do not call `flush()` from an ISR.
- Serialize access when the SPI bus is shared with a DAC or another peripheral.
- Update the brightness buffer first and then perform one complete `flush()`.
- Use `Silent` when startup latency matters. The visible policies intentionally block.
- Keep BLANK connected whenever a visually silent and deterministic power-up is required.
