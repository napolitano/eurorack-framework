# MCP3208

## Overview

Eight-channel 12-bit successive-approximation ADC with SPI interface.

| Property | Framework view |
|---|---|
| Manufacturer | Microchip |
| Framework library | `eurorack-driver-mcp3208` |
| Public header | `<eurorack/drivers/adc/mcp3208.hpp>` |
| Primary class | `Mcp3208` |
| Hardware resources | SPI; CS; external analog reference |
| Maturity | Unreleased Alpha |

## Why this device is supported

The device covers a recurring embedded/Eurorack hardware task while remaining independently selectable. Projects that do not use it do not need to add its library or transitive dependencies.

## Hardware function

Consult the manufacturer data sheet for absolute maximum ratings, electrical characteristics, package pinout, timing limits and PCB-layout requirements. Framework documentation explains the software contract; it does not replace the data sheet or electrical design review. Eurorack signals must never be connected directly unless the external circuit keeps every device pin within its permitted voltage and current range.

## Key electrical and protocol characteristics

| Characteristic | Practical meaning |
|---|---|
| Inputs | Eight single-ended inputs or four differential input pairs |
| Resolution | 12 bit |
| Conversion interface | SPI transaction starts and returns one conversion result |
| Reference | External VREF defines the ADC input full scale |
| Input behavior | Source impedance and acquisition settling directly affect accuracy |
| Readback | Conversion data is returned in the same SPI transaction |

## Pin and signal integration

| Device signal | Framework dependency | Integration requirement |
|---|---|---|
| CLK / DIN / DOUT | `SpiBus` | Full-duplex transfer is required |
| CS/SHDN | `DigitalOutput` | Must delimit every conversion frame |
| CH0..CH7 | Analog front end | Inputs must remain inside the permitted range |
| VREF | External reference | Decouple close to the device and keep digital return currents away |

## Framework operation sequence

Call `read()` with a channel and input mode. The driver validates the channel, builds the control bits, performs one three-byte transfer, and decodes the 12-bit result into the caller-provided value.

## Framework support

Supported: All eight channels, single-ended and differential command modes, 12-bit result extraction and configurable SPI clock.

Not currently supported: Device-specific diagnostics, readback or uncommon special modes are not exposed unless listed above.

## Dependencies and ownership

The library owns no global scheduler and performs no dynamic dispatch beyond the framework I/O interfaces. The consuming application owns the bus and GPIO adapter objects and must ensure that their lifetime exceeds the driver lifetime. Bus arbitration follows the `IoResult` contract; calls may report Busy or transport errors.

Hardware resources: **SPI; CS; external analog reference**. Sharing a bus is supported only when each device has independent selection/latch signals and all transfers occur through the same bus abstraction. Driver calls are foreground operations unless the application and platform adapter explicitly document ISR safety.

## Minimal code example

```cpp
#include <eurorack/drivers/adc/mcp3208.hpp>

Mcp3208 adc(spi, chipSelect);
std::uint16_t sample = 0U;
const auto result = adc.read(0U, sample);
```

The variables representing buses, pins, delays and canvases are platform adapters supplied by the application. Check every returned `IoResult` in production code.

## Integration notes

1. Verify supply voltage, logic thresholds, reference voltage and output compliance against the exact part and package.
2. Initialize selection, latch, blanking or enable pins to a safe inactive state before enabling the peripheral.
3. Keep bus transactions outside time-critical ISRs unless a platform-specific contract explicitly permits them.
4. Add decoupling and analog filtering according to the manufacturer recommendations and the required noise performance.
5. Validate the implementation on target hardware; native tests prove protocol formation and state handling, not analog performance.
